# i18n — ICU4J + `lang/` directory

**Date:** 2026-05-18  
**Scope:** All 15 Thymeleaf templates; full EN/FR coverage; ICU4J plural support

---

## Goal

Replace all hardcoded French strings in Thymeleaf templates with `#{}` message expressions,
backed by `lang/messages.properties` (EN default) and `lang/messages_fr.properties` (FR).
Add ICU4J for proper plural rules. Move existing messages files under `lang/`.

---

## Directory structure

```
src/main/resources/
  lang/
    messages.properties         ← EN (default/fallback)
    messages_fr.properties      ← FR
```

`application.properties`:
```properties
spring.messages.basename=lang/messages
spring.messages.encoding=UTF-8
```

---

## ICU4J integration

`build.gradle.kts`:
```kotlin
implementation("com.ibm.icu:icu4j:76.1")
```

New `@Configuration` class `I18nConfig`:
```java
@Bean
public MessageSource messageSource() {
    var source = new ResourceBundleMessageSource() {
        @Override
        protected MessageFormat createMessageFormat(String msg, Locale locale) {
            return new com.ibm.icu.text.MessageFormat(msg, locale);
        }
    };
    source.setBasenames("lang/messages");
    source.setDefaultEncoding("UTF-8");
    return source;
}
```

ICU plural syntax in properties:
```properties
obs.predicate.count={0,plural,=0{No predicate}one{# predicate}other{# predicates}}
```

## Plural usage

The only confirmed plural case is `obs.predicate.count`.  
Template before:
```html
<span th:if="${pb.predicates.empty}">Aucun prédicat</span>
<span th:unless="${pb.predicates.empty}" th:text="${pb.predicates.size()} + ' prédicat(s)'"></span>
```
Template after:
```html
<span th:text="#{obs.predicate.count(${pb.predicates.size()})}"></span>
```

The same pattern applies in `global-process-rules.html` for rule predicates.

## `th:text` vs `th:utext`

Most keys use `th:text` (HTML-safe). A small set of keys embed inline HTML markup and must use `th:utext` instead:

| Key | Reason |
|-----|--------|
| `danger_zone.description` | Contains `<strong>7 days</strong>` |
| `admin.process_rules.description` | Contains `<strong>all users</strong>` |
| `admin.process_rules.add.note` | Contains `<strong>`, `<code>` inline |

All three are static, admin-only strings (no user-supplied content), so `th:utext` is safe here.
